# Чек-лист аудита

> Использовать при любом аудите (процессов, документов, подразделений, конкурентов).

## Перед аудитом
- [ ] Определить цель: что именно аудитируем и зачем
- [ ] Прочитать ВСЕ файлы по теме (не выборочно)
- [ ] Прочитать COMPANY.md (контекст бизнеса)
- [ ] НЕ ПИСАТЬ ВЫВОДОВ пока не прочитано всё

## Проверки
- [ ] Полнота: все ли аспекты охвачены?
- [ ] Актуальность: данные свежие или устаревшие?
- [ ] Противоречия: нет ли конфликтов между документами?
- [ ] Пробелы: что отсутствует но нужно по контексту?
- [ ] Риски: что может навредить бизнесу?

## Формат отчёта
1. КРИТИЧЕСКОЕ — требует немедленного действия
2. ВАЖНОЕ — нужно сделать в ближайших сессиях
3. РЕКОМЕНДАЦИИ — улучшения, не блокеры
4. ПРОВЕРЕНО И ОК — что в порядке (обязательно!)

Каждое утверждение = ссылка на источник (файл, строка, факт).

## После аудита
- [ ] Протокол "Проверено" в конце
- [ ] Цифры перепроверены
- [ ] Результат сохранён в файл (reports/ или соответствующая папка)
- [ ] Если были исправления — один повторный проход по этому чек-листу перед коммитом. Третий круг = стоп, показать владельцу
