Проверь код: $ARGUMENTS

Если аргумент — имя файла, проверь этот файл. Если не указан — спроси что проверить.

## Критерии проверки

### 1. Читаемость
- Понятные названия переменных и функций?
- Функции не длиннее 50 строк?
- Нет ли дублирования кода?

### 2. Ошибки
- Обрабатываются ли ошибки (try/except, if err, set -e)?
- Нет ли очевидных багов (неинициализированные переменные, бесконечные циклы)?
- Правильно ли закрываются ресурсы (файлы, соединения)?

### 3. Безопасность
- Нет ли секретов в коде (пароли, ключи, токены)?
- Проверяются ли входные данные?
- Нет ли инъекций (SQL, shell, HTML)?

### 4. Надёжность
- Что будет при таймауте или обрыве сети?
- Есть ли retry для внешних вызовов?
- Логируются ли ошибки?

## Формат ответа

По каждому критерию:
- Критерий — всё хорошо
- Критерий — замечание: [что и где]
- Критерий — проблема: [что, где, как исправить]

В конце:
```
Оценка: X/10
Главное исправить: [1-2 самых важных пункта]
```

Если файл большой (>200 строк) — проверяй по частям, не пропускай.
