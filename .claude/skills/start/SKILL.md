Начало рабочей сессии.

Шаги:
1. Прочитай HANDOFF.md — что было в прошлой сессии
2. Прочитай ERRORS.md — какие ошибки известны
3. Покажи кратко:
   - Что было сделано в прошлой сессии (3-5 пунктов)
   - Какой следующий шаг по плану
   - Есть ли нерешённые ошибки
4. Спроси: "Работаем по плану или есть другая задача?"

## Понедельник — автоматическая еженедельная уборка

Если сегодня понедельник (или первая сессия на неделе), ПЕРЕД показом сводки выполни уборку:

### Диагностика
- Запусти `bash weekly_check.sh` — собери список проблем

### Уборка (делай автоматически, не спрашивай)

1. **Пустые отчёты** — если файл в reports/ < 100 байт, удалить (мусор от сбойных запусков)
2. **ERRORS.md** — если ошибка помечена как решённая и старше 30 дней, предложить удалить (не удалять молча — показать список)
3. **Дубли и устаревшее в документации** — проверить HANDOFF.md, CLAUDE.md, COMPANY.md:
   - Устаревшие данные (сравнить с реальными файлами)
   - Дублирование между файлами
   - Неактуальные пункты в приоритетах

### Отчёт
Покажи одним блоком:
```
Еженедельная уборка:
- Отчёты: [что сделано]
- Документация: [что найдено]
- Ошибки: [статус]
```

Потом продолжай обычный /start (сводка, план, вопрос).
