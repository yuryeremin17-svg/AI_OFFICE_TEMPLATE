# AI_OFFICE_TEMPLATE — Заготовка ИИ-офиса для клиентов

## Что это
Продукт "ИИ-офис под ключ" для топ-менеджеров и собственников бизнеса.
Клиент получает ссылку → вставляет в Claude → получает работающий офис за 10 минут.

## Статус
Готов к первому реальному тесту. Все файлы созданы, 4 аудита пройдены, продажные материалы подготовлены.

## Последняя сессия — 13 марта 2026
- 4 аудита: структура, dev/клиент разделение, перекрёстные ссылки, путь клиента
- Продажные материалы: 3 формата (сообщение, пост, лендинг) x 3 версии (универсал, SMM Дубай, мед топ)
- Go-to-market стратегия: 3 тарифа, воронка, дорожная карта до $100K/мес, экспертная критика
- Ценообразование: Разведка $2500, Штаб $5000+$1000/мес, Партнёр $10000+$2000/мес
- Следующий шаг: реальный тест онбординга на одном из двух клиентов

## Архитектура (решения сессии 58)

**Подход:** Модульный конструктор. Ядро общее + скиллы комбинируются под клиента.
Первые два клиента — тест-кейсы для проверки модулей.

**10 функций ИИ-офиса:**
1. Знать клиента и бизнес → COMPANY.md + PROFILE.md
2. Думать правильно, проверять себя → CLAUDE.md (мозг)
3. Помнить контекст между сессиями → HANDOFF.md
4. Выполнять задачи → skills/ (SKILL.md — инструкции для Claude, без bash)
5. Управлять департаментами и проектами → departments/ + projects/
6. Хранить результаты работы → reports/
7. Учиться и улучшаться → improvements.md в каждом скилле
8. Связывать задачи в цепочки → workflows в CLAUDE.md
9. Показать ценность сразу → демо-скилл после онбординга
10. Легко войти → onboarding/ONBOARDING_PROMPT.md

**Структура:**
```
AI_OFFICE_TEMPLATE/
├── CLAUDE.md                ← мозг (плейсхолдеры, заполняется онбордингом)
├── COMPANY.md               ← бриф компании (плейсхолдеры, заполняется онбордингом)
├── PROFILE.md               ← профиль клиента (плейсхолдеры, заполняется онбордингом)
├── HANDOFF.md               ← контекст между сессиями
├── DEV.md                   ← этот файл (удаляется после онбординга)
├── departments/             ← оргструктура (по мере работы)
│   └── DEPT_TEMPLATE.md
├── projects/                ← проекты (по мере работы)
│   └── PROJECT_TEMPLATE.md
├── skills/                  ← инструменты
│   ├── daily_brief/SKILL.md
│   ├── research/SKILL.md
│   ├── content/SKILL.md
│   ├── qa/SKILL.md
│   ├── scout_market/SKILL.md
│   └── smm_calendar/SKILL.md
├── reports/                 ← результаты работы (подпапки по скиллам)
├── sales/                   ← продажные материалы (удаляется после онбординга)
└── onboarding/
    └── ONBOARDING_PROMPT.md ← точка входа клиента
```

**Порядок разработки:**
1. CLAUDE.md (шаблон мозга)
2. COMPANY.md + PROFILE.md (шаблоны брифов)
3. HANDOFF.md
4. Скиллы универсальные (daily_brief, research, qa)
5. Скиллы профильные (content, scout_market, smm_calendar)
6. Шаблоны департаментов и проектов
7. Онбординг-промпт (последний — когда знаем что он собирает)
8. Тест на двух профилях

**Два тест-кейса:**
- Управленец в медицинской компании (РФ, Mac) — препараты, обучение представителей, открытие представительств в арабских странах
- SMM-маркетолог у брокера недвижимости (Дубай)
