# Онбординг ИИ-офиса

## Что вставляет клиент (скопировать и отправить в Claude Code)

```
Настрой мне персональный ИИ-офис.

Репозиторий: https://github.com/yuryeremin17-svg/AI_OFFICE_TEMPLATE

Склонируй его, задай мне вопросы и настрой под меня. Покажи первый результат.
```

---

## Инструкция для Claude (выполнять после клонирования)

### Общие правила онбординга
- Говори на языке клиента (определи по первому сообщению)
- Тон: деловой, тёплый, без панибратства. Ты настраиваешь рабочий инструмент
- Не хвали без причины, не используй маркетинговые фразы
- Весь онбординг — одна сессия, ≤10 минут
- Не задавай вопросов, на которые клиент уже ответил

---

### ШАГ 0: Скачивание и подготовка

```bash
git clone https://github.com/yuryeremin17-svg/AI_OFFICE_TEMPLATE.git ~/Documents/AI_OFFICE
cd ~/Documents/AI_OFFICE
git remote remove origin
```

Скажи клиенту: "Офис скачан. Задам несколько вопросов, чтобы настроить под вас. Займёт 10 минут."

---

### ШАГ 1: Быстрый старт (5 вопросов → демо)

Задай ВСЕ 5 вопросов ОДНИМ сообщением:

```
Чтобы настроить офис, мне нужно знать:

1. Как вас зовут, какая должность, город?
2. Какая компания, чем занимается? (отрасль, продукт/услуга, рынки)
3. Какие рабочие задачи отнимают больше всего времени?
4. Ведёте соцсети для бизнеса? Если да — какие платформы?
5. На каком языке работаем? (русский / English / оба)
```

После ответа клиента:

**1. Заполни COMPANY.md** — из ответов на вопросы 2, 3:
- Название, отрасль, продукт, рынки — что сказал клиент
- Остальные поля (конкуренты, цели, боли, специфика) — оставь пустыми, заполним в такте 2

**2. Заполни PROFILE.md** — из ответов на вопросы 1, 3, 5:
- Имя, должность, город
- Приоритеты: из ответа на вопрос 3
- "Чего хочу от ИИ-офиса": из ответа на вопрос 3
- Остальное (бэкграунд, стиль решений, слабости) — заполним в такте 2

**3. Подбери скиллы** — по ответам на вопросы 3 и 4:

| Если клиент... | Подключить |
|----------------|-----------|
| Управляет людьми/бизнесом | daily_brief, research, qa, scout_market |
| Ведёт соцсети/контент | daily_brief, research, qa, content, smm_calendar |
| Упоминает конкурентов/тренды/рынок | + scout_market (независимо от роли) |
| Оба варианта | все 6 скиллов |

**4. Заполни CLAUDE.md** — он уже в корне, заполни плейсхолдеры:
- Заполни секцию "Кто я" из PROFILE.md
- Заполни "Язык" из ответа на вопрос 5
- Заполни таблицу "Активные скиллы" — подобранные скиллы с описанием
- Заполни "Цепочки" — в зависимости от скиллов:
  - Управленец: scout_market → daily_brief → решение
  - Маркетолог: research → content → qa → публикация; smm_calendar → content
  - Оба: все цепочки
- Приоритеты: из ответа на вопрос 3

**5. Обнови HANDOFF.md** — первая запись:
```
## Последняя сессия
**Дата:** [сегодня]

**Что сделано:**
- Настроен ИИ-офис: COMPANY.md, PROFILE.md, CLAUDE.md
- Подключены скиллы: [список]

**Следующий шаг:**
- Дополнить профиль (такт 2)
```

**6. Запусти демо-скилл:**
- Если управленец → daily_brief (утренняя сводка на основе COMPANY.md)
- Если маркетолог → content (черновик поста на основе бренда из COMPANY.md)
- Сохрани результат в reports/

**7. Покажи результат** и скажи:
"Вот ваш первый результат. Офис работает. Теперь уточним детали — это займёт ещё 5 минут, и офис будет работать точнее."

### ШАГ 2: Дозаполнение (уточняющие вопросы)

Задай ОДНИМ сообщением:

```
Теперь уточним, чтобы офис работал точнее:

1. Кто ваши основные конкуренты? Чем вы от них отличаетесь?
2. Какие цели на этот год? А на ближайший квартал?
3. Что больше всего мешает в работе? Где узкие места?
4. Как вы принимаете решения — быстро по интуиции или взвешенно по данным?
5. Есть ли что-то, что ИИ-ассистент НЕ должен делать? (например: не принимать решения за вас, не писать клиентам)
```

После ответа:

**1. Дополни COMPANY.md:**
- Конкуренты (из вопроса 1)
- Цели стратегические и операционные (из вопроса 2)
- Боли и проблемы (из вопроса 3)
- Специфика отрасли (что можешь определить из контекста)

**2. Дополни PROFILE.md:**
- Стиль принятия решений (из вопроса 4)
- "Как со мной работать" — "Не делать" (из вопроса 5)
- Известные слабости (из вопроса 3 — что мешает)

**3. Обнови CLAUDE.md:**
- Приоритеты — уточни из новых ответов

---

### ШАГ 3: Завершение онбординга

Скажи клиенту:

```
Офис настроен. Вот что у вас есть:

📋 Профиль и бриф компании — Claude знает ваш контекст
🧠 Правила работы — верификация, протоколы, стиль
🛠 Скиллы: [перечислить подключённые]
📊 Первый результат: [что было показано в демо]

Как пользоваться:
- В начале дня скажите "сводка" — получите план на день
- Скажите "исследуй [тема]" — глубокий анализ с проверкой
- Скажите "проверь [текст]" — редактура и улучшение
[+ специфичные для подключённых скиллов]

В конце каждой сессии скажите "сохрани" — я обновлю историю и сделаю бэкап в iCloud.
В следующий раз продолжим с того же места.
```

**4. Очистка:** удали папки onboarding/, sales/ и файл DEV.md — клиенту они не нужны.

**5. Обнови HANDOFF.md** — финальная версия:
```
## Последняя сессия
**Дата:** [сегодня]

**Что сделано:**
- Настроен ИИ-офис (онбординг завершён)
- Заполнены: COMPANY.md, PROFILE.md, CLAUDE.md
- Подключены скиллы: [список]
- Первый результат: [что показали]

**Решения:**
- Язык: [какой]
- Скиллы: [какие и почему]

**Следующий шаг:**
- [первый приоритет клиента из его ответов]
```

**6. Первое сохранение:**
```bash
git add -A && git commit -m "Онбординг завершён"
rsync -a ~/Documents/AI_OFFICE/ ~/Library/Mobile\ Documents/com~apple~CloudDocs/AI_OFFICE_BACKUP/
```
Скажи: "Всё сохранено. Бэкап в iCloud сделан."

### Чеклист: что должно быть заполнено после онбординга

**Обязательно (всё к концу сессии):**
- [ ] COMPANY.md: название, отрасль, продукт, рынки, конкуренты, цели, боли
- [ ] PROFILE.md: имя, должность, роль, стиль решений, приоритеты, "не делать"
- [ ] CLAUDE.md: плейсхолдеры заполнены, скиллы подключены, цепочки прописаны
- [ ] HANDOFF.md: первая запись
- [ ] reports/: первый результат демо-скилла
- [ ] git: локальный commit
- [ ] iCloud: бэкап в ~/Library/Mobile Documents/com~apple~CloudDocs/AI_OFFICE_BACKUP/

**Допустимо оставить пустым (заполнится в первых сессиях):**
- PROFILE.md: бэкграунд (подробный), сильные стороны, рабочий режим
- COMPANY.md: структура, клиенты (каналы, средний чек), специфика отрасли (если не очевидна)
- departments/ — создаются когда клиент начнёт работать с направлениями
- projects/ — создаются под конкретные проекты
- improvements.md в скиллах — создаются автоматически после первого использования скилла

**Когда заполняется "допустимо пустое":**
- Сессия 2-3: Claude сам предлагает "Давайте дополним профиль" когда видит пробелы
- При первом запуске скилла: если не хватает данных — Claude спрашивает и дозаполняет
